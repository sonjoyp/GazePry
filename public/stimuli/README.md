# D7 stimulus packs

Real image files for the recognition-probe task, described by
`manifest.json` and generated by `scripts/make_stimuli.py`.

```bash
python scripts/make_stimuli.py            # (re)generate everything
python scripts/make_stimuli.py --check    # validate an existing pack
```

| Set | Ships as | Ready to collect? |
|---|---|---|
| **E1** | 24 Julia-set fractals | **Yes** |
| **E2** | 16 generated marks | **No — placeholders** |
| **E3** | 16 generated marks | **No — placeholders** |

The task page **blocks** E2 and E3 while any item is marked `"placeholder": true`.
That is deliberate rather than a warning: those experiments measure familiarity
the participant brought with them, so a cohort run against stand-ins produces
data that cannot be salvaged, and by the time anyone reads a warning the
participants have gone home.

---

## Why E1 is generated and E2/E3 are not

E1's validity depends on the participant having **no prior exposure**, so that
familiarity is created only by the study phase and the ground truth is exact.
Photographs of real things fail that test — everyone has seen a beach, a dog, a
keyboard — and the "novel" items would carry uncontrolled pre-existing
familiarity that no counterbalancing can remove. Richly detailed *novel abstract*
images are the standard stimulus class for recognition-memory work for exactly
this reason.

The generator enforces two properties that a naive fractal dump does not have:

- **Detail.** Escape-time is histogram-equalised and the interior is coloured by
  an orbit trap, because the obvious mapping produces a dark image with a thin
  bright rim — low variance and little for a participant to actually recognise.
- **Mutual distinctiveness.** Every pair is checked in *colour* to differ by at
  least 22 mean absolute RGB levels, and candidates are resampled until that
  holds. If a "novel" tile looks like one the participant studied, the
  familiarity contrast is contaminated in a way counterbalancing cannot undo.
  The achieved minimum is recorded as `minPairDistance` in the manifest.

E2 and E3 measure *naturally acquired* familiarity, so they need the real thing
and you have to supply it.

## Installing real assets for E2 / E3

### 1. Prepare the images

- **E2:** service logos or homepage screenshots. Pick a set whose real-world
  penetration **varies** — the `tier` field (`high` / `medium` / `low`) records
  your expectation and drives the "high-salience items only" fallback analysis.
- **E3:** topic or article cards. Keep to health, finance, legal, and civic.
  **Do not add protected characteristics** (sexual orientation, religion,
  immigration status): the method would apply, the demonstration does not need
  them, and including them turns a privacy paper into an ethics problem. A test
  asserts the shipped E3 categories, so adding one will fail the suite.

Requirements: **at least 600 × 450 px** (below that the image is upscaled into
the tile and loses the detail recognition depends on), 4:3-ish aspect, PNG or
JPEG. `--check` enforces the size and reports anything missing.

### 2. Drop them in

Replace the files in `e2/` (or `e3/`) keeping the same names, or use new names
and update `manifest.json`.

### 3. Mark them as real

Set `"placeholder": false` on each item you replaced, and record where it came
from:

```json
{
  "id": "mail",
  "label": "Mailbox",
  "file": "e2/mail.png",
  "kind": "brand",
  "tier": "high",
  "placeholder": false,
  "source": "https://example.com/press/logo.svg",
  "licence": "trademark, nominative fair use — research only, not redistributed",
  "retrieved": "2026-07-22"
}
```

`source`, `licence`, and `retrieved` are not validated by the tooling, but a
paper that reproduces a stimulus figure needs them and reconstructing provenance
after the fact is miserable. Fill them in while you still remember.

### 4. Verify

```bash
python scripts/make_stimuli.py --check
```

Regenerating with `make_stimuli.py` (no `--check`) **overwrites** `e2/` and
`e3/` with placeholders again. Keep your real assets backed up outside the repo,
or generate into a scratch directory with `--out`.

## Licensing and what gets committed

`.gitignore` excludes `e2/` and `e3/` image files. Third-party logos and
screenshots are someone else's trademarked or copyrighted material, and this
repo is GPLv3 and intended for public release — vendoring them would relicense
material that is not yours to relicense.

- `manifest.json` **is** committed (it is your design, and describes the pack).
- `e1/` **is** committed (generated here, reproducible from seeds, no third-party
  rights).
- `e2/` and `e3/` images are **not** committed. Distribute them alongside the
  artifact separately, or ship the manifest plus a fetch script.

For the paper: displaying a logo to identify the service it denotes is normally
nominative use, but that is a judgement for you and your institution, not
something this README settles.
